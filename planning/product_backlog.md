# Product Backlog

Priority: High 

User Story: 

**As a patient, I want to schedule a telehealth appointment, so that I can receive medical care without having to travel to a clinic.** 

Acceptance Criteria:

•	Patients can choose an appointment that is available.

•	The system displays a confirmation after the appointment has been booked.

Priority: High

User Story: 

**As a patient, I want to be able to complete my online forms so that I can save time before my appointment.** 

Acceptance Criteria:

•	Patients are able to successfully submit completed forms.

•	The information is securely stored.  

Priority: High

User Story: 

**As a healthcare provider, I want to be able to view my appointment schedule, so that I can be able to prepare for upcoming patient visits.**

Acceptance Criteria:

•	Providers can view their daily schedule.

•	The schedule automatically updates when an appointment is changed.

Priority: Medium

User Story: 

**As an administrator, I want to be able to manage user accounts and their permissions, so that only authorized users are able to access sensitive information.**

Acceptance Criteria:

•	Administrators can create, update and disable user accounts.

•	User permissions are based on assigned roles.

Priority: Medium

User Story: 

**As a patient, I want to receive appointment reminders so that I do not miss my scheduled appointment.**

Acceptance Criteria:

•	Appointment reminders are sent before their visits.

•	The reminder includes the appointment data and time.

Priority: Low

User Story: 

**As a healthcare provider, I want my appointments to be synced with my calendar so that I can manage my schedule better.**

Acceptance Criteria:

•	Calendar events are updated automatically. 

•	Duplicate appointments are prevented.
