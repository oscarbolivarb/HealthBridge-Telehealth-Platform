# Sprint 1 Plan

## Sprint Goal

The goal for the sprint 1 is to develop the basic features needed for patients to be able to schedule an appointment and complete forms while creating a secure foundation for the rest of the project.

---

## Backlog Items Selected

-	Scheduling patient appointment
-	-	Online patients forms 
-	Provider appointment schedule 
-	User account and role management
  
---

## Team Assignments 

Since this project is being completed individually, one team member will complete all the project’s responsibilities.

Team Role & Responsibilities

Project Manager: plan the sprint 1, organize the project’s documentation and track the overall progress.

Lead Developer: develop the appointment scheduling feature and patient forms.

System Architect: design the system structure and plan any future integrations with external systems.  

QA Analyst: test complete features and verify that they meet the projects requirements

UX Designer: design friendly screens and improve accessibility for different type of users.

---

## Known Risks and Dependencies

### Risks 

-	Integrating APIs with third party calendars can require additional development and testing.
-	Meeting HIPAA security requirements can increase development time.
-	Accessibility features must be tested throughout development to ensure that all users can successfully use the platform.

### Dependencies

-	In order for the role base access to be implemented a user authentication must be completed.
-	The appointment scheduling depends on the calendar integration
-	The patient’s forms require a secure database in order to store the submitted information.
