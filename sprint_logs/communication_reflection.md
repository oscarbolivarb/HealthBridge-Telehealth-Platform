Workflow & Communication Reflection 

Since the milestone was completed independently, there was no communication challenges. The main challenge was to make sure to keep track of all of the project requirements and to check that all required files was placed in the correct folder. 

This was addressed by reviewing the instructions and complete one task at the time, GitHub was used to organize the project files and clear commit messages was used to document each update. This made it easier to review the completed work and to verify that every required file had been added to the repository, before it was submitted. 

For future sprints one improvement could be updating the sprint board immediately after each task when it is complete instead of waiting to the end since this could provide a more accurate view of the progress and also help keep the project organized from the beginning to the end.
