Leadership Reflection 

During the sprint, leadership focused on staying organized and making sure that each task was completed before moving to the next task and since the project was completed independently, all of the responsibility was important including planning the work, setting priorities and keeping the project on schedule. 
One leadership action that supported the sprint was reviewing the assignment requirements before starting the work; this way it could be organized into smaller tasks and no required field would end up being missing. 
Normally, this project would have been divided into responsibilities with others but this was a solo project, so the sprint plan was intended to complete the documentation, update the sprint board and review the GitHub repository before submitting the assignment. 
By following this process it helped keep the work organized throughout the sprint. For the next sprint, leadership could be improved by updating the sprint board more frequently as tasks are being completed, this would make it easier to monitor the progress throughout the sprint and at the same time identify any remaining work.
