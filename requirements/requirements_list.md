# System Requirements 

## Functional Requirements 

### FR-1 Appointment Scheduling 

The system should allow patients to be able to schedule, reschedule and cancel their telehealth appointments through the platform.

### FR-2: Patient Forms

The systems should allow patients to complete and submit a secure electronic form before their appointment.

### FR-3: Role Base Access 

The system should provide different levels of access for patients, healthcare providers and administrators based on their roles.

### FR-4: Calendar Integration 

The system should be able to synchronize patient’s appointments with third party calendar applications, this will help patients stay organized.    

---
## Non-Functional Requirements

### NFR-1: Security 

The system should be able to protect the patient’s information by using encryption and following HIPAA security requirements.

### NFR-2: Accessibility

The platform should be able to follow accessibility guidelines so that users who are visually impaired can use the system easily.

### NFR-3: Performance 

The system should load important pages within three seconds during normal use.

### NFR-4: Availability

The platform 99.9% of the time should be available, except when there is maintenance scheduled.
